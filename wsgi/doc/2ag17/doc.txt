第八週摘要報告
===

以協同方式寫Wiki

本文目的在於讓學生利用開發的wiki與應用快速原型的協作式寫作的開發方法。而本文也由定性數據收集和分析方法的來評價。
最後，對於協作方式來寫作的方式，其影響極其討論協議到組別與軟件開發方面的考慮和教學相關的要求問題
研究方法 wiki的協作式寫作，基於設計的研究包括以下四個步驟：
1. 研究的現狀與認識目的並審查與wiki的協作式寫作相關的問題。
2. wiki將用於促進以協同方式寫作來設計，參與和群體互動。
3. 使用多種方法收集其經驗數據。
4. 通過系統的評價分析，並通過各種方法收集資料。分析，設計，實施和評價是相互共存的。而缺點提出於每個週期提出，重新設計，重新實現，並重新評估。
維基應用域教育在協同方式寫作方法而成的Wiki是適用於多種情況，期望得到的wiki應用在學習投資效益與協同方式來寫作。它十分靈活的，足以適應的專業條件。更具體地，它的應用領域包括不同級別的更高教育，從研究生到社會教育。除了協同寫作與學科相關，維基可以為一些應用程序的開發系統來使用，如產生教材，網路上互相評論，並收集數據於一種項目。


結論和未來研究方向
1. 這項結果不能限制於狹小的研究的範圍，即使調查結果反映了那些報導中的一致性研究文獻。
2. 用來判斷以不同方式來呈現協同，尤其是同組評議，這擁有極高的教育價值與分析性思維，並參與交流。可以促進團體間的合作和時間，wiki打開了協同寫作上新的視野，而小組互動絕不是容易的事情，解決了技術，教學和文化各種問題。
未來的工作將更顯的協同的重要性，wiki以此設計為基礎下更加精進。重要的是，它更進一步影響高等教育對於wiki的使用。最後，它也進一步實踐於系統評論的基礎

期末報告
===

摘要

  用現在現有的技術，來編寫出一份能夠在近端及遠端上的協統套件，近端的繪製零件轉為STL檔，上傳到我們主要協統的網站，讓我們的產品能夠在網站中檢視，也能用遠端來達成自動零件組立。
  此套件不但能夠把我們的產品放入網站讓所有客戶能夠不用點擊下載，就能選擇所需的零組件，也能在網站上點擊連結直接下載所需的零件等。

概論
1.1協統設計探討
   在網路沒有普及時，機械設計基本上大多數是以小組進行，一組大約少數人，也有僅一人設計。但在至今網路普及的時代裡，能夠達到遠端組別共同設計同一個專案，不但能夠增加速度，也能夠確保此專案的可行度，大大提升現代人們科技的進步。
   現在已經有些公司利用遠端協統來增快專案進度，例如:在美國正常上班時間時是印度休息時間，此時專案在印度進行，當美國下班時間時，剛好可以交給印度繼續進行專案設計，這樣的確能夠提升專案的進度，所以現在雖然在台灣中小型企業裡在協統都已近端為主，但如果想擴大公司的營運，就必須以少資本少時間的方式盡量擴展，就必須先利用網路上的免費套件來做有效的營運。

1.2協統、非協統
   協統對於工程上最基本上是能夠同時讓兩位工程師用CAD來設計專案，也能夠同時讓設計者、客戶、公司等能夠一起分享訊息，以增快彼此的需求，以最快的方式達到我們所需要的產品。

TeamProject
  本套件能夠使零件在網站上讓遠端用戶者檢視，還能修改尺寸，已達到每位客戶的需求，倘若有需要可提供零件下載。以CMSimply來進行內容的管理，檔案擇放在Bitbucket、copy連結提供下載，且能讓所有參與此專案的設計者，能夠透過網路來溝通，也就能讓專案的進行速度更快。

目前具備技術
1.在近端利用 Creo 繪製零件
2.利用遠端 Pro/Web.Link 程式修改近端 Creo 零件尺寸
3.將近端零件轉為 STL, 上傳至遠端 CMSimply 網站中檢視
4.在近端利用手動組立零件
5.在近端利用遠端網站上的 Pro/Web.Link 進行自動組立
6.在近端與雲端建立 Dokuwiki 網站
7.利用手動或半自動方式管理 Dokuwiki 的使用者
8.在近端與雲端建立 CMSimply 網站
9.利用 Github 與 Bitbucket 管理所有文字資料的版次
10.利用近端 Leo Editor 與可攜程式系統管理所有資料
11.利用近端環境產生 pdf 與 html 格式報告資料

套件使用流程
1.	在Creo繪製零件後轉回stl檔可在網路上檢視
2.組合圖在creo自動組立。
3.依照使用者可自行選擇尺寸。
4.倘若有問題可透過cmsimply回饋

結論
    在這20世紀，網路帶來巨大的改變，現在有很多關於網路上的技術、程式語言都逐漸的成熟，甚至更加的便利，加快軟、硬體上的設計。這些都脫離不了網路帶給現代的人能夠更快傳遞訊息，在19世紀初傳遞訊息上可能要花好幾天才能辦到，擇現在只需花幾秒即可。
    協統設計是每位設計師都會遇到的，無論是在機械或電子領域等，都需要這些技術才能在日後的競爭環境中脫穎而出。
    這次套件是屬於簡單的協統套件，但基礎流程也讓我們學到不少，想在公司裡有個一席之地，或許可以選擇學習之前原有的技術與公司的前輩上司相互競爭，但也能利用此協統方式研究出對公司有良好的貢獻，使公司能夠更加快速度完成每項作業，也許這樣會使自身在公司的地位中會有個不可取代的位子。
